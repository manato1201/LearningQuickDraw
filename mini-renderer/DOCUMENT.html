1. プロジェクト概要

Apple Macintosh (1984) のグラフィクスAPI QuickDraw の描画アルゴリズムを 現代C++で再実装したCPUソフトウェアレンダラー。
流体エフェクト・映像描画最適化の基礎研究プロジェクト Phase 1 として構築。

項目内容
言語C++17
描画方式CPU ソフトウェアレンダリング（GPU使用なし）
ウィンドウSDL2 (FetchContentで自動取得)
参考文献The Art of QuickDraw (Apple, 1984)
目的描画パイプライン基礎理解 / 流体エフェクト最適化の土台

3. アルゴリズム詳解

QuickDraw: 0501DDALine.p

3-1. DDA ライン描画

DDA (Digital Differential Analyzer) は直線描画の基本アルゴリズム。 傾き slope = dy/dx を1ステップずつ加算してピクセル座標を求める。

原理

水平方向が主軸 (|dx| >= |dy|):
  slope = dy / dx

  for x = x0 to x1:
    putPixel(x, round(y))
    y += slope

垂直方向が主軸 (|dy| > |dx|):
  slope = dx / dy

  for y = y0 to y1:
    putPixel(round(x), y)
    x += slope

実装

float slope = (float)dv / (float)dh;
float fy    = (float)y0;
for (int x = x0; x <= x1; x++) {
    putPixel(x, (int)std::round(fy), color);
    fy += slope;
}

3-2. 固定小数点最適化

float演算 (除算・round) を整数演算 (乗算・ビットシフト) に置き換えて高速化。 QuickDrawがMotorola 68000アセンブリで実践した最適化手法。

16.16 固定小数点形式

32bit整数を上位16bit(整数部) + 下位16bit(小数部) に分割

3.5  =>  3 * 65536 + 32768  =  0x00038000

変換:
  float -> fixed  :  (int)(value * 65536)  または  value << 16
  fixed -> int    :  value >> 16            (round()の代替)
  スロープ計算   :  (dy << 16) / dx        (除算は初期化時1回のみ)
  加算           :  fixed_a + fixed_b       (通常の整数加算)

実装比較

// --- float版 (遅い) ---
float slope = (float)dv / (float)dh;  // float除算
float fy    = (float)y0;
for (int x = x0; x <= x1; x++) {
    putPixel(x, (int)std::round(fy), color);  // round() コスト大
    fy += slope;
}

// --- 固定小数点版 (速い) ---
int slope = (dv << 16) / dh;  // 除算は1回だけ
int fy    = y0 << 16;
for (int x = x0; x <= x1; x++) {
    putPixel(x, fy >> 16, color);  // >>16 = round()代替
    fy += slope;                    // 整数加算のみ
}

3-3. スラブ三角形ラスタライズ

三角形を水平なスキャンライン（スラブ）に分解し、 各行の左端・右端X座標を長辺と短辺の交差点から求めて塗りつぶす。 現代GPUのラスタライザと本質的に同じ処理。

処理の流れ

      * (x0, y0)      <-- top
     /|
    / |  長辺 (y0->y2)
   /  |
  * --+-------        <-- 各スキャンラインで横塗り (スラブ)
  |   |
  |   |  長辺続き
  *---*               <-- bottom (x2, y2)
(x1,y1)

1. 頂点をY順にソート: top / mid / bottom
2. 長辺 (v0->v2) と短辺 (v0->v1, v1->v2) を特定
3. for y = y0 to y2:
     xa = 長辺上のX  =  x0 + (x2-x0) * (y-y0) / totalH
     xb = 短辺上のX
     for x = xa to xb: putPixel(x, y)
現代の拡張 (QuickDrawにはなかった概念)

3-4. Zバッファ（深度バッファ）

複数の三角形が重なるとき、手前のピクセル（Z値が小さい）だけを描画する。 ピクセルと同解像度の float 配列で深度を管理。

データ構造

Framebuffer:
  m_pixels[W * H]  : ARGB8888 色バッファ
  m_depth[W * H]   : float    深度バッファ  (初期値: FLT_MAX)

深度テスト

bool testAndSetDepth(int x, int y, float depth) {
    float& slot = m_depth[y * m_width + x];
    if (depth < slot) {   // 新しいZが既存より手前?
        slot = depth;      // バッファ更新
        return true;       // 描画許可
    }
    return false;          // 描画スキップ (奥にある)
}

スキャンライン内Z補間

// 辺の端点でZ値を補間 (スラブと同じ手順でZも計算)
float za = z0 + (z2 - z0) * t;      // 長辺上のZ
float zb = z1_edge;                  // 短辺上のZ

// ピクセルごとにZ補間
float frac = (x - xa) / (xb - xa);
float z    = za + (zb - za) * frac;

if (fb.testAndSetDepth(x, y, z)) {
    fb.setPixel(x, y, color);
}

毎フレームのリセット

fb.clear(0xFF111111);  // 色バッファをクリア
fb.clearDepth();        // 深度バッファを FLT_MAX にリセット

3-5. 透視投影

焦点距離: f = 1 / tan(fov / 2)

投影:
  px = (x * f) / z
  py = (y * f) / z

スクリーン座標:
  sx = px *  W/2  +  W/2   (中心 = 画面中央)
  sy = py * -H/2  +  H/2   (Y軸反転: スクリーン下方向がプラス)

深度値:
  depth = z  (小さい = カメラに近い = 手前)
QuickDraw: Graf3D Pitch() / Yaw() / Roll() / 0601RotatingCube.p

3-6. 3D回転行列（Pitch / Yaw / Roll）

各軸の回転行列

X軸 Pitch:          Y軸 Yaw:            Z軸 Roll:
| 1   0    0  |     | c  0  s |          | c -s  0 |
| 0  cos -sin |     | 0  1  0 |          | s  c  0 |
| 0  sin  cos |     |-s  0  c |          | 0  0  1 |
(c = cos(angle), s = sin(angle))

合成と適用

// 毎フレーム角度を更新
pitch += 0.010f;
yaw   += 0.020f;
roll  += 0.005f;

// 行列合成
Mat4 rot = Mat4::rotationX(pitch)
         * Mat4::rotationY(yaw)
         * Mat4::rotationZ(roll);

// 全頂点に適用
for (int i = 0; i < 8; i++) {
    Vec3 v = rot.transform(CUBE_VERTS[i]);
    projected[i] = perspectiveProject(v, W, H, 60.0f);
}

3-7. リージョンクリッピング

矩形の範囲外ピクセルを棄却する。ハードウェアのシザーテストと同等。

void putPixel(int x, int y, uint32_t color) {
    if (m_clipEnabled) {
        if (x < m_clipX || x >= m_clipX + m_clipW) return;
        if (y < m_clipY || y >= m_clipY + m_clipH) return;
    }
    m_fb.setPixel(x, y, color);
}
